# 拓展 3：图形化界面 — 技术说明

## 概述

在 CLI 命令行的基础上，构建了一个交互式校园地图可视化界面，使用 **HTML Canvas + 纯 JavaScript** 实现，零外部依赖，双击即可运行。

## 技术选型

| 选项 | 说明 |
|------|------|
| **渲染** | HTML5 Canvas（HiDPI 支持，2x/3x 高分屏锐利） |
| **布局** | 力导向图（Force-Directed Graph），30 轮迭代后节点位置自然反映连接关系 |
| **最短路径** | 内置 Dijkstra 算法，DIST/TIME 双模式 |
| **关键节点** | 基于度中心性（Top-K 高度节点） |
| **交互** | 节点/边的点击悬停响应、侧边栏实时信息面板 |

## 如何运行

双击 `campus_map.html`，浏览器自动打开，即可交互操作。

## 必须展示项（对照验收标准）

| 验收要求 | 实现状态 | 说明 |
|---------|---------|------|
| 节点位置 | ✅ 完成 | 力导向布局（节点斥力+边弹力），节点分布自然不重叠 |
| 边连接，区分 open/closed | ✅ 完成 | open 边绿色，closed 边红色虚线 |
| 最短路径在图上高亮 | ✅ 完成 | 下拉框选起终点 + DIST/TIME 模式 → 点击查询 → 路径节点金色、边金色加粗 |
| 关键节点/边在图上标识 | ✅ 完成 | 关键节点品红色+发光阴影，关键边界限颜色虚线 |

## 功能亮点（自主发挥，1 分）

| 亮点 | 说明 |
|------|------|
| **HiDPI 高清渲染** | Canvas 物理分辨率 = CSS 尺寸 × devicePixelRatio，2x/3x 屏幕字迹清晰 |
| **力导向布局** | 20 轮迭代后节点自然分散，边密集区域局部集群明显 |
| **边权重标注** | 每条边上标注距离（xxxm） |
| **节点详情面板** | 点击节点 → 右侧显示 ID、display_name、类别、度数、邻居列表 |
| **边详情面板** | 点击边 → 右侧显示两端节点号、distance、walk_time |
| **悬停高亮** | 鼠标靠近节点时节点放大+白色边框 |
| **操作重置** | Reset 按钮一键清除所有高亮 |
| **数据来源于 make_my_test.py** | 60 节点 120 边全连通随机图，展示效果最饱满 |

## 数据来源

图形展示的数据由项目根目录下的工具生成：

```bash
cd tools
python make_my_test.py random 60 120 42
```

生成的 `places.csv` 和 `roads.csv` 直接嵌入 HTML 的 JavaScript 数据区。

## 技术栈

- HTML5 Canvas（HiDPI 支持）
- 原生 JavaScript（无框架、无外部库）
- Dijkstra 最短路径算法
- 力导向图布局算法（斥力+弹力+迭代阻尼）
- `window.devicePixelRatio` 高清适配
