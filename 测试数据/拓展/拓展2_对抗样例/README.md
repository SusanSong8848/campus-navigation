# 对抗样例说明：全等权多环路图

## 为什么这组数据特殊？

这组数据有 12 个节点、18 条边，**所有边的 distance 完全相同（都是 10），walk_time 完全相同（都是 1）**。

图中有大量环路：A01-A02-A04-A09-A11-A08-A06-A04-A02-A01 形成复杂的网状结构。

## 它能卡住哪类错误实现？

### 1. Dijkstra 未处理等权边多路径

如果 Dijkstra 实现在同权边上依赖某个固定遍历顺序（如按 `place_id` 字典序遍历邻接边），不同实现的路径会完全不同。一个错误的实现可能在等权边构成的"菱形"结构中选择错误的中间节点。

这组数据检验：`SHORTEST A01 A12 DIST` 能否正确找到路径（所有路径总距离相同，都是多个 10 之和），以及路径是否包含重复节点（错误的 Dijkstra 可能因回边产生环路）。

### 2. 关键节点/边暴力枚举的超时风险

图中有大量环路，每个节点都属于至少两个环。暴力枚举关键节点/边时，如果 BFS 没有正确使用 `visited` 集合去重，会因环路陷入无限循环。同时，大量等权边会让 `AllEdges` 和 `GetAdjacentEdges` 的高频调用压力暴露潜在的性能问题。

### 3. MST 等权边的排序稳定性

所有边权相等时，Kruskal 算法对边的排序结果可能因排序算法稳定性或字典序规则而不同。如果代码依赖某组特定的边选择顺序，这组数据会让它原形毕露。

## 我的程序如何正确处理？

| 对抗点 | 我的处理方式 |
|--------|-------------|
| 等权边多路径 | Dijkstra 使用优先队列按全局最短距离松弛，与邻接边遍历顺序无关 |
| 环路防循环 | BFS 使用 `visited` 哈希集标记已访问节点，确保每个节点只入队一次 |
| 等权边 MST | Kruskal 对所有边按 distance 排序，同权边按字典序稳定排，MST 边数为 V-1 |
| 关键节点/边性能 | BFS 屏蔽实现不修改原图，惰性删除只是 `continue` 跳过，不增加额外复杂度 |
